⭐ STEP 1: START EVERYTHING
═══════════════════════════════════════════════════════════════════════════════

🖥️  Windows - Run this in PowerShell:
    docker-compose up -d --build


🐧 Linux/Mac - Run this in terminal:
    docker-compose up -d --build


(Wait 10 seconds for services to start)


⭐ STEP 2: OPEN THE SWAGGER API DOCS
═══════════════════════════════════════════════════════════════════════════════

Open this in your browser (copy the URL):
    http://localhost:8000/docs

You can upload CVs here and test API endpoints interactively!


⭐ STEP 3: MONITOR REAL-TIME CV PROCESSING (MOST IMPORTANT!)
═══════════════════════════════════════════════════════════════════════════════

🔴 COPY & PASTE THIS COMMAND in a NEW terminal/PowerShell window:

    docker-compose logs -f worker --tail=100 | grep -E "(STAGE|TASK|ERROR|SUCCESS|Duration)"


Windows (PowerShell) version - if grep doesn't work:
    docker-compose logs -f worker --tail=100 | findstr /I "STAGE TASK ERROR SUCCESS Duration"


This shows you CV processing in REAL-TIME:
✓ Shows each stage of processing (Load, PDF, LLM, Database)
✓ Shows timing information
✓ Shows errors if something fails
✓ Shows when processing completes successfully


⭐ STEP 4: UPLOAD A CV
═══════════════════════════════════════════════════════════════════════════════

Option A: Using Swagger UI (Easiest)
   1. Go to http://localhost:8000/docs
   2. Find the "/upload" endpoint
   3. Click "Try it out"
   4. Choose a PDF file
   5. Click "Execute"

Option B: Using terminal command

   Windows (PowerShell):
   ────────────────────────────────────────────────────────────────────────────
   $file = "C:\Path\To\Your\CV.pdf"
   $form = @{ file = Get-Item $file }
   Invoke-WebRequest -Uri "http://localhost:8000/upload" -Method Post -Form $form


   Linux/Mac (cURL):
   ────────────────────────────────────────────────────────────────────────────
   curl -X POST -F "file=@backend/data/cvs/your_cv.pdf" http://localhost:8000/upload
   (Place PDFs in backend/data/cvs/ — split bundle outputs go to backend/data/cvs_split/ first; see README.)

⭐ OPTIONAL: CELERY MONITORING DASHBOARD
═══════════════════════════════════════════════════════════════════════════════

To see active tasks in a visual dashboard:
   http://localhost:5555

(This shows task queue status in real-time)


⭐ OPTIONAL: CLEAN UP INCOMPLETE RECORDS BEFORE RUNNING
═══════════════════════════════════════════════════════════════════════════════

Before processing a batch, flush any incomplete records:

Windows (PowerShell):
   (Invoke-WebRequest -Method POST http://localhost:8000/api/admin/flush-incomplete).Content


Linux/Mac (cURL):
   curl -X POST http://localhost:8000/api/admin/flush-incomplete


⭐ STOP ALL SERVICES
═══════════════════════════════════════════════════════════════════════════════

When done, run:
   docker-compose down
